Arquitecturas web

Introducción

Se puede definir arquitectura web como la elección, planificación, diseño e implementación de los elementos necesarios para el funcionamiento de una aplicación web:

  • Hardware

  • Sistemas operativos y servidores

  • Aplicación en servidor

Cuya finalidad principal es servir las aplicaciones a los clientes en cualquier momento.

Modelo cliente/servidor

El modelo cliente servidor es la principal y primera arquitectura utilizada en las aplicaciones web, aunque no es exclusivo de estas. Por ejemplo, los sistemas operativos suelen utilizarlo para la gestión de las ventanas (servidor X Windows), o servidores de correo y de base de datos, entre muchos otros.

Teniendo dos elementos principales:

  • Cliente: Se encarga de realizar las peticiones al servidor enviando la información necesaria para obtener la información.

  • Servidor: Se encuentra esperando peticiones de clientes (puede atender a más de un cliente al mismo tiempo). En el caso más sencillo obtiene el recurso (fichero) y lo envia al cliente, en caso más general, a partir de la información que facilita el cliente y de la que dispone el servidor, se fabrica la respuesta y se envia al cliente.

En el caso de la web los clientes son navegadores web y los servidores pueden ser servidores web o servidores de aplicaciones, existiendo otros servidores que dan apoyo al servidor web/aplicación, como servidor de base de datos o servicios de directorio.

Existen otras arquitecturas de red, dependiendo de los roles y las funcionalidades:

  • Peer to peer (P2P). En este caso los elementos son tanto clientes como servidores.

    Entre las ventajas destacan:

    • Escalabilidad

    • Tolerancia a fallos

    • Descentralización

    • Privacidad

    • Equilibro de carga

    Y las desventajas:

    • Complejo

    • Menos seguro

    • Más tráfico

  • Sistema distribuido. Colección de sistemas/servidores autónomos, sin memoria compatida que interactua entre sí para realizar una determinada labor común, viéndose desde el exterior como un único elemento.

    Ventajas:

    • Economía

    • Crecimiento:Escalabilidad

    • Alto rendimiento

    • Fiabilidad

  • Editor/subscriptor. En este caso, no es el cliente el que solicita la petición, sino que se subscribe al servidor o a servicios de este y el servidor (editor) envia a todos los subscritores la información cuando se produce el evento al que se ha subscrito.
